# 指令组

|               指令               | 需要的权限 |      指令说明      |
| :------------------------------: | :--------: | :----------------: |
| `/todo add 待办事项 [user_name]` |  操作待办  |    添加待办事项    |
|    `/todo rm <n> [user_name]`    |  操作待办  |   删除待办事项 n   |
|   `/todo done <n> [user_name]`   |  操作待办  |   完成待办事项 n   |
|    `/todo clear [user_name]`     |  操作待办  |    清空待办列表    |
|     `/todo show [user_name]`     |  查看待办  |    查看待办列表    |
| `/todo chmod <mod> [user_name]`  |  修改属性  | 更改待办的权限范围 |
|     `/todo how [user_name]`      |  查看待办  | 让 AI 给待办做规划 |

# 功能规划

- [x] `/todo add 待办事项`：【操作】添加待办事项

- [x] `/todo rm <n>`：【操作】删除待办事项 n

- [x] `/todo done <n>`：【操作】完成待办事项 n

- [x] `/todo show [user_name]`：【查看】查看待办事项列表

- [x] `/todo clear`：【操作】清空待办事项列表

- [x] 用户可控制待办事项的权限范围 `/todo chmod <mod> [user_name]`：

  权限分为三级：查看待办、操作待办、修改属性。

  权限对**待办所属用户**均开放，只针对待办所属用户之外的用户进行设定，因此部分 `<mod>` 解读如下：

  **000/0：仅私人可见可操作；**

  100/4：公开可查看待办；

  110/6：公开可查看、操作待办；

  001/1：公开可修改属性；

  111/7：公开可查看、操作待办，可修改属性；

- [x] 可在配置中设置用户待办的默认权限

- [ ] 定时给用户发送他的待办列表，提醒他完成；可在配置中设置是否定时，定时几点等

- [x] 调用大模型，对用户的 TODO 进行一个规划
